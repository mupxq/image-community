# 影像社区 — 产品纲要

> 图文内容的AI共创社区，围绕影像内容展开，以AI与用户共创为核心，形成创作、阅读与社交结合的社区体验。

---

## 一、产品定位

一句话：**让任何人都能参与内容创作，和别人一起创作比一个人更有趣。**

核心逻辑：大量中层创作者有想法但缺乏从零完成作品的能力。当他们看到别人的作品时，往往觉得"我有更好的点子"——我们用这个冲动驱动他们行动。AI补上技能短板，降低创作门槛；共创机制让每个人的贡献都有意义。

与大模型的关系：**共生而非竞争。** AI能力越强，用户在平台上创作的东西越好，平台价值越高。我们不是工具，工具会被替代；我们是社区和内容资产，这不会被替代。

---

## 二、核心机制

### 2.1 分支共创

- 用户发起一个创作（漫画/短剧），发布后其他用户可基于此续写
- 每次续写产生一个新分支，形成**树状创作结构**（类似平行宇宙）
- 分支前的内容不可修改，分支后的部分属于续写者
- 所有为某条故事线贡献内容的人，都是该作品的**共同作者**

### 2.2 双模创作

| 模式 | 说明 |
|------|------|
| 自己创作 | 用户上传图片，自行编辑分镜、场景描述、对白，完全掌控内容 |
| AI创作 | 用户只需描述故事梗概，选择画面风格和页数，AI一键生成分镜，用户可逐页修改后发布 |

两种模式在续写场景下同样适用。

### 2.3 作品形态

- **即时分享作品**：类似小红书/抖音，短小、可单人闭环，即做即发
- **连续型作品**：类似短剧/连载漫画，有完整叙事性，需主创持续输出，社区质量更高

## 三、社交体系

社交是产品的核心，不是附属功能。

### 3.1 关系类型

| 关系 | 描述 |
|------|------|
| 创作者 ↔ 创作者 | 共创群聊、合作邀请、创作讨论 |
| 创作者 ↔ 读者 | 粉丝关注、更新通知、打赏互动 |
| 读者 ↔ 读者 | 基于同一作品线的讨论、兴趣圈层 |

### 3.2 互动方式

- 评论、弹幕
- 作品投票（对分支进行推荐）
- 创作挑战（共创题目，如AI接龙、同人续写）
- 私聊与共创群聊

---

## 四、书架系统

用户的阅读管理中心：

- 收藏作品，分类为：想读 / 在读 / 已读完
- 显示阅读进度（已读页/总页数）
- 从作品详情页一键加入书架
- 支持标记状态切换和移除

---

## 五、商业模式

- **内容分成**：参考番茄小说模式，共创作品按贡献分成——只要某条创作线盈利，整条线上的创作者都受益
- **虚拟打赏**：读者对优秀作品/作者赞助
- **广告与带货**：优质内容对接品牌曝光合作
- **IP授权**：平台对优秀作品进行商业授权（游戏/影视/出版）
- **Web3路径**：链上版权存证，NFT概念，透明的链上分成机制——这是产品壁垒

---

## 六、竞争壁垒

| 壁垒 | 说明 |
|------|------|
| 网络效应 | 不是单人创作社区，用的人越多内容越好，吸引更多人加入 |
| 内容资产 | 每条创作线都可能成为IP，且持续衍生同人作品 |
| 创作关系链 | 用户在平台上积累的共创关系和作品分支，迁移成本极高 |
| 数据飞轮 | 用户进行AI创作不断反馈数据，强化平台的AI生成能力 |
| Web3版权机制 | 链上确权和透明分成，传统平台无法快速复制 |

## 七、用户画像

### 轻创作者（核心用户）
- 拍摄爱好者、新锐插画师、AI绘图玩家、短视频创作者
- 有表达欲，但不一定有完整叙事能力
- 渴望曝光和创作伙伴

### 潜在读者-创作者（增长用户）
- 喜爱影像内容、动漫、剧情设定
- 参与评论、弹幕、投票、提供灵感
- 有尝试意愿，被"创作门槛"劝退
- **AI创作模式专门为这群人设计**

## 八、产品架构

### 功能模块总览

```
影像社区 APP
├── 发现页（首页）        — 瀑布流展示作品，分类筛选
├── 书架                  — 收藏管理，阅读进度追踪
├── 创作（中心按钮）       — 自己创作 / AI创作 / 续写
├── 消息                  — 私聊、共创群聊
├── 我的                  — 个人主页、作品管理、共创记录
│
├── 作品详情/阅读页        — 分镜浏览、共创者署名、评论区
├── 创作树                — 树状可视化展示作品分支关系
└── 续写页                — 基于已有作品创建新分支
```

### 核心数据结构

```
User（用户）
Work（作品）→ parent_work_id 形成树状结构，root_work_id 指向根节点
WorkPage（分镜页）→ 支持标记是否AI生成
Contributor（共创者）→ 记录每个作品的所有参与者及角色
Bookmark（书架收藏）→ 阅读状态 + 进度
Conversation / Message（会话/消息）→ 支持私聊和共创群聊
```

## 九、实现路径

### 阶段一：工具验证（当前）
- 目标：AI照片风格化 + 基础创作发布
- 形态：独立新APP
- 验证指标：功能使用频率、生成图质量、社交媒体传播性

### 阶段二：社区化
- 目标：将工具型产品社区化，创作完成后一键发布，外部分享转向内部分享
- 核心理念：**一切都是作品**（想法、草稿、评论都算）
- 引入共创机制和社交功能
- 验证指标：用户分享率、留存率、活跃度

### 阶段三：内容生态
- 目标：推出长内容创作分享（连载漫画/短剧），形成独立社群氛围
- 验证指标：创作者数量、故事线阅读量与评论、用户活跃

### 远期目标
- 全球最大内容共创社区
- 验证指标：作品数量、IP数量、创作者数量、用户活跃

## 十、关键决策记录

| 决策 | 结论 | 理由 |
|------|------|------|
| 新APP vs 系统相册迭代 | 新APP | 产品定位清晰、可针对性宣传、架构自由度高 |
| 冷启动策略 | 投入运营资金 | 用户密度和意愿匹配靠运营解决，不是产品瓶颈 |
| 内容质量控制 | 后期处理 | 推荐算法是规模化后的事，初期不阻塞 |
| 版权机制 | Web3链上确权 | 核心壁垒，非技术选型而是产品战略 |
| 创作模式 | 自己创作 + AI创作双模 | 覆盖有能力的创作者和只有想法的创作者 |
